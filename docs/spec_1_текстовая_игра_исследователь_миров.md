# SPEC-1 — Текстовая игра «Исследователь миров»

## Background

Вы хотите однопользовательскую текстовую игру с исследованием процедурно расширяемых миров. Игровой цикл — чат: Гейм‑Мастер (ГМ, модель Gemini Pro) описывает текущее место, предметы и возможные выходы; игрок отвечает, указывая действие/направление. Если путь реалистичен в рамках сеттинга/атмосферы/жанра и структуры мира, ГМ перемещает игрока или создает новую локацию и связи, затем возвращает описание.

Ключевые идеи:
- На стартовом экране — список сохранённых миров и кнопка «Создать мир».
- При создании мира задаются/рандомизируются три начальные установки: **сеттинг, атмосфера, жанр**. По ним ГМ генерирует центральную ноду и заготовки связей.
- Мир хранится локально между сессиями как **граф JSON**: ноды‑локации с уникальным ID, `location_name`, описанием, предметами и связями с другими нодами.
- Предметы — отдельные JSON‑объекты; могут лежать в локациях или находиться в **инвентаре игрока** (перенос между локациями сохраняется).
- Игрок имеет **имя и описание персонажа**; ГМ описывает мир **от первого лица** этого героя, опираясь на три установки мира.
- Алгоритм хода ГМ: сверка намерения с известными выходами → при необходимости правдоподобное порождение ноды/ребра → перемещение игрока → ответ с описанием локации и доступных путей (включая возврат).
- Выход «Завершить путешествие» возвращает к стартовому экрану со списком миров.

Этот документ будет уточняться по разделам (Requirements → Method → Implementation → Milestones → Gathering Results). На каждом шаге зафиксируем минимально реализуемую, но конкретную архитектуру, пригодную для немедленной разработки.
